# 🤖 AI Agent 生产级开发提示词：NGPM 多视角流量分析系统

## 一、 全局状态管理与交互状态机（State Machine）

为了让高级运维在自主排障时拥有“无缝滑行”的流畅感，前端必须维护一个强关联的状态机。AI Agent 在编写组件时须严格遵循以下全局状态变量：

### 1. 全局状态机定义 (React/TypeScript)

```typescript
interface GlobalFilterState {
  currentView: 'global' | 'link' | 'app' | 'ip' | 'tunnel'; // 5大视角切换
  timeRange: { start: number; end: number; granularity: '1s' | '100ms' }; // 时间切片与微突发粒度
  selectedProbe: string[]; // 探针过滤
  pinnedFilters: {
    type: 'link' | 'app' | 'ip' | 'protocol' | 'session';
    value: string;
    label: string;
  }[]; // 全局 Pin 过滤面包屑
  activeOnionLayer: 'DNS' | 'TCP' | 'TLS' | 'L7' | 'DB' | null; // 洋葱模型当前聚焦层
  selectedEntityId: string | null; // 当前视角下左侧列表选中的具体 ID（如某条链路名或服务名）
  isDrawerOpen: boolean; // 右侧会话详情抽屉开关
  activeSessionId: string | null; // 激活的会话五元组 ID
}

```

### 2. 全局点击交互响应规约（Interaction Outcomes）

* **【动作】点击顶部 5 大视角切换**：修改 `currentView`。页面主体框架重组，但必须**继承**当前的 `timeRange` 和 `pinnedFilters`，自动刷新新视角下的数据。
* **【动作】点击左侧列表条目**：修改 `selectedEntityId`。右侧看板所有图表触发 loading，带入该实体 ID 作为 Filter 重新请求数据。
* **【动作】在时序图上“拖拽框选”时间段**：修改 `timeRange.start` 和 `end`。下方“协议层”、“Top Talker”立刻变为该**时间切片内的临时统计值**。
* **【动作】右键点击任意图表中的实体（如 Top Talker 中的某个 IP）**：弹出气泡菜单 `[固定为全局筛选]`。点击后向 `pinnedFilters` 数组 `push` 一个对象，顶部筛选栏生成对应的**面包屑标签**，全局视图立即收紧至该过滤范围。

## 二、 5 大视角界面重组与假数据路由字典

当 `currentView` 切换时，AI Agent 须执行不同的组件挂载（Mount）与数据映射：

```
+------------------------------------------------------------------------------------+
| 顶部导航: 全局总览 | 按链路 (Selected) | 按应用 | 按IP/网段 | 按隧道                      |
+------------------------------------------------------------------------------------+
| [左侧: 链路列表]         | [右侧主看板]                                              |
| ■ BJ↔SH 专线 (利用率84%) | 1. 核心指标: 吞吐量、TCP重传率、乱序率、SRTT时延             |
| □ DC1↔DC2 互联           | 2. 趋势图: 框选触发 100ms 粒度微突发捕获 ⚡                   |
| □ DMZ 出口               | 3. Top Talker: 自动拆分为 [入向流量 | 出向流量]              |
+------------------------------------------------------------------------------------+

```

### 1. 全局总览视角 (`'global'`)

* **界面布局**：隐藏左侧侧边栏，中央主看板全屏化。
* **假数据填充语义**：
* 总吞吐：`4.82 Gbps`，总包率：`682 kpps`，重传率：`0.42%`。
* 中层默认展示 **“AI 检测到的事件河流”**（支持点击收起，收起后下方“服务通信健康矩阵”和“协议层健康”上移）。



### 2. 按链路视角 (`'link'`)

* **界面布局**：左侧挂载 **`<LinkList />`**，右侧为主看板。
* **点击交互结果**：点击左侧列表的 `BJ↔SH 专线` $\rightarrow$ 顶部探针栏自动将 `WAN-Edge-01` 和 `DMZ-01` **置灰（不可选）**，高亮该专线相关的探针。
* **数据组件重组**：
* **核心指标**：变更为 `吞吐(4.82Gbps) | TCP重传率(1.4%) | 乱序率(0.8%) | SRTT时延(38ms)`。
* **Top Talker 表格**：列结构重组为 `[客户端→服务端 | 入向流量 | 出向流量 | 重传包数]`，明确方向性。



### 3. 按应用视角 (`'app'`)

* **界面布局**：左侧挂载 **`<AppServiceList />`**（置顶 `order-svc`、`auth-svc`）。
* **数据组件重组**：
* 中层变更为 **洋葱模型依赖性流水线 (`<OnionPipeline />`)**。
* 默认状态下，下方时序图展示该应用的吞吐趋势。



### 4. 按 IP/网段视角 (`'ip'`)

* **界面布局**：左侧挂载 **`<IPNetTree />`**（支持 `/24` 或业务安全域聚合，如 `核心生产网段`、`办公网段`）。
* **数据组件重组**：核心指标切换为 `并发连接数 | 新建连接率(cps) | PPS(包率)`。图表聚焦于 Top Talker 矩阵，用于快速定位内网大文件拷贝或异常扫描。

### 5. 按隧道视角 (`'tunnel'`)

* **界面布局**：左侧挂载 **`<TunnelList />`**（展示 VNI ID、VXLAN/GRE 隧道名称）。
* **数据组件重组**：指标卡片并列展示 `外层 Outer IP 质量 (丢包/抖动)` 与 `内层 Inner IP 业务性能`。
* **智能诊断解包逻辑**：若内层 TCP 重传率高，指标栏下方悬浮标红提示：`“检测到内层大包重传，外层路径 MTU 溢出，建议检查分片（Fragmentation）设置”`。

## 三、 核心高阶组件深度细化与开发规范

### Component 1: 洋葱模型依赖性流水线 (`<OnionPipeline />`)

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  1. DNS解析  │ ──> │  2. TCP传输  │ ──> │  3. TLS握手  │ ──> │  4. L7协议层 │ ──> │ 5. 数据库/后 │
│ P95: 320ms(红│     │重传: 1.4%(黄)│     │握手: 18ms(绿)│     │ 5xx: 24%(红) │     │ 慢查询: 3/min│
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘

```

* **级联受害者遮罩（Victim Mask）交互结果**：
* 当左侧选择 `order-svc`，系统检测到 `TCP传输` 层异常（重传率 1.4%），AI Agent 须在下游的 `TLS握手`、`L7协议层`、`数据库` 卡片上自动叠加半透明遮罩类（`.opacity-40 .pointer-events-none`）。
* 并在卡片间连接箭头上渲染虚线动画，悬浮提示：*“由于底层 TCP 丢包，上层业务时延被级联放大，请优先处理网络层。”*


* **点击卡片 Focus Mode 交互**：
* **点击【1. DNS解析】** $\rightarrow$ 下方时序图切换为 `DNS解析延迟与失败率曲线`，右侧列表切换为 `NXDOMAIN 异常域名排行`。
* **点击【5. 数据库】** $\rightarrow$ 下方时序图切换为 `SQL 平均耗时时序图`，右侧列表切换为 `慢 SQL 语句排行榜/具体文本`。



### Component 2: 微突发捕获时序图 (`<MicroBurstTrafficChart />`)

* **技术规范**：基于 ECharts 实现。
* **微突发虚线标记（MarkLine）**：
* 当数据源中存在微突发事件（如 `microBurstCount > 0`），在 ECharts 的 `series` 中动态配置 `markLine`，在突发绝对时间轴上绘制**垂直红色虚线**，并在顶部渲染闪电图标 ⚡。


* **高阶 Zoom-in 交互结果**：
* **点击闪电图标** 或 **在趋势图上拖拽框选该区域** $\rightarrow$ 触发 `onBrushEnd` $\rightarrow$ 全局状态 `timeRange.granularity` 变更为 `'100ms'` $\rightarrow$ 向后端发起高精点请求 $\rightarrow$ 时序图重绘，展现毫秒级无采样尖峰，击穿“平均值陷阱”。



### Component 3: 深度会话抽屉与高级 Stevens Graph (`<SessionDiagnosticDrawer />`)

当用户在任何视角的 会话/Top Talker 列表中点击具体流（如 `10.20.5.14:52341 → mysql-cluster:3306`）时，修改 `isDrawerOpen = true`，从右侧滑出此组件。

```
+--------------------------------------------------------------------------+
| 会话详情: 10.20.5.14:52341 -> mysql-cluster:3306                         |
+--------------------------------------------------------------------------+
| [五维时延分解瀑布图]                                                      |
| 网络RTT(12ms) | 服务端处理(280ms)====== | 重传阻塞(154ms)=== | 零窗口(84ms)=  |
+--------------------------------------------------------------------------+
| [高级专家面板: TCP 序列号与时延联动图 (Stevens Graph)]                       |
| Sequence Number (Bytes)                                                  |
| ^                                                                        |
| |  .  .  .  .  .  .  .  . [横向平直线: 滑动窗口满/零窗口]                  |
| | /  /  /  /  /  /                                                       |
| |/  /  /  /  /  [垂直红叉: 快重传 / RTO丢包]                              |
| +──────────────────────────────────────────────────────────> Time (ms)   |
+--------------------------------------------------------------------------+
| [PCAP 智能切片下载]                                                       |
| [ 📥 下载全量 PCAP ]  [ ✂️ 仅下载异常发生前后 100MB 报文 (推荐) ]           |
+--------------------------------------------------------------------------+

```

* **时延五维分解瀑布图 (`<WaterfallChart />`)**：
* 必须严格对应洋葱模型指标，横向堆叠：

$$\text{Total Latency} = \text{Network RTT} + \text{Server Process} + \text{Data Transfer} + \text{Retransmission Block} + \text{Zero Window Block}$$




* **专家级 Stevens Graph (`<StevensChart />`)**：
* X 轴为绝对时间（毫秒级），Y 轴为 TCP Sequence Number。
* **故障特征图形化渲染规约**：
1. **丢包/重传**：当捕获到相同 Seq Num 报文时，在对应坐标上渲染**红叉**。
2. **接收端窗口冻结**：当抓到零窗口公告（Zero Window App），且 Seq Num 曲线变成**水平横线**持续 $>50\text{ms}$ 时，该段横线高亮为**橙色**，直接做实客户端应用层消费太慢、缓冲区被塞满的根因。




* **PCAP 智能切片下载交互**：
* 点击“下载全量 PCAP”：请求全量文件。
* 点击“仅下载异常前后 100MB 报文”：请求带上时间窗口参数：`/api/pcap/download?sessionId=xxx&startTime=${triggerTime - 30s}&endTime=${triggerTime + 30s}`，极大节省下载时间。
